# Bootstrap Astra Linux for Ansible

## 📌 Назначение

Данный документ описывает процесс подготовки системы Astra Linux Orel для работы с Ansible.

По умолчанию Astra Linux не полностью совместима с Ansible, так как:

* отсутствует подходящая версия Python;
* отсутствуют необходимые Python-библиотеки;
* стандартные Ansible-модули (например, apt) могут работать некорректно.

В связи с этим требуется предварительный этап bootstrap.

---

## 🎯 Цель bootstrap

Подготовить систему к выполнению Ansible playbook'ов:

* установить современную версию Python;
* настроить интерпретатор для Ansible;
* обеспечить возможность выполнения модулей;
* минимизировать использование raw в дальнейшем.

---

## ⚙️ Общая схема

Чистая Astra Linux
        ↓
Установка Python 3.9 (скрипт)
        ↓
Настройка ansible_python_interpreter
        ↓
Работа Ansible playbook'ов

---

## 🚀 Шаг 1. Подключение к машине

Подключение осуществляется по SSH:

ssh user@<ip_address>

Требования:

* доступ по SSH уже настроен;
* пользователь имеет права sudo.

---

## 🔧 Шаг 2. Передача скрипта установки Python

Скопировать скрипт на удалённую машину:

scp scripts/install-python.sh user@<ip_address>:/tmp/

---

## 🐍 Шаг 3. Установка Python

Выполнить на удалённой машине:

ssh user@<ip_address>
sudo bash /tmp/install-python.sh

---

## 📍 Результат установки

После выполнения скрипта:

* Python установлен в:

/opt/python/3.9

* Интерпретатор доступен по пути:

/usr/local/bin/python3.9

---

## ⚙️ Шаг 4. Настройка Ansible

В inventory необходимо указать интерпретатор:

[astra_hosts]
astra ansible_python_interpreter=/usr/local/bin/python3.9

---

## 🧪 Шаг 5. Проверка

Проверить доступность Python:

ansible astra -m ping

Ожидаемый результат:

astra | SUCCESS => {
    "ping": "pong"
}

---

## ⚠️ Особенности Astra Linux

### 1. Ограниченная совместимость с Ansible

Стандартные модули могут работать некорректно из-за:

* версии системного Python;
* отсутствия зависимостей (python3-apt).

---

### 2. Использование raw

В проекте для Astra Linux используется:

* raw для установки пакетов (apt-get);
* минимизация зависимости от Python-модулей Ansible.

---

### 3. Различия с Ubuntu

| Компонент  | Ubuntu   | Astra               |
| ---------- | -------- | ------------------- |
| Python     | готов    | требуется установка |
| apt module | работает | ограниченно         |
| bootstrap  | не нужен | обязателен          |

---

## 🔒 Безопасность

* установка выполняется через sudo;
* используется локальный скрипт (контролируемый код);
* не используются сторонние репозитории;
* Python устанавливается в /opt, не затрагивая системный.

---

## 📦 Состав скрипта

Скрипт install-python.sh:

* устанавливает зависимости для сборки Python;
* скачивает исходный код Python 3.9;
* компилирует и устанавливает в /opt/python/3.9;
* создаёт симлинк /usr/local/bin/python3.9;
* устанавливает pip.

---

## 🚫 Ограничения

* установка занимает время (сборка из исходников);
* требуется доступ в интернет;
* не используется пакетный менеджер для Python;
* не выполняется автоматическое обновление Python.

---

## 🎯 Вывод

Bootstrap-этап является обязательным для Astra Linux и позволяет:

* обеспечить совместимость с Ansible;
* унифицировать процесс автоматизации;
* использовать единые playbook'и для разных ОС;
* минимизировать platform-specific проблемы.

После выполнения bootstrap система готова к применению ролей:

* base
* users
* security
